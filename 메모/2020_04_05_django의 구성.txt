### 장고의 구성


## 1. config
> django-admin startproject config .
을 통해 만들어진 메인 프로젝트 앱 폴더
config .은 현재 폴더에 config라는것으로 프로젝트를 만들자는 의미

# 1.1 urls.py
url을 설정하는데 사용

# 1.2 settings.py
서버를 세팅하는 단계에서 사용
나중에는 세팅을 추가하거나, 사용자 입맛에 맞게 설정하는데 사용
처음에는
ALLOWED_HOST
AINSTALLED_APPS
이 두 부분만 건드려도 충분함.

# 1.3 그외(asgi.py, wsgi.py, __init__.py)
__init__.py는 폴더 안쪽의 py들을 모듈로 사용한다고 선언.
나머지 두 파일은 사이트 배포단계에서 사용함.

#----------------------------------------------

## 2. mysite
> django-admin startapp mysite
를 통해 만들어진 앱 폴더

나중에 만약 mysite안에 여러가지 기능등을 구현 해놓고 다른 장고 앱에 복사하려면
이 폴더만 들고가면 됨.

# 2.1 admin.py
admin페이지에 이 앱을 추가하는데 사용됨.

# 2.2 apps.py
이것이 앱이라는것을 명시해 주는 파일

# 2.3 models.py
***매우 중요함***
장고는 MTV(Model-Template-View)라고 해서, 일단 모델부터 먼저 만들어야
View까지 감. 모델부터 잘못 만들게 되면 View까지도 못감.

# 2.4 tests.py
사이트를 테스트할때 사용됨.

# 2.5 views.py
사이트 내 페이지를 구성하는 역할을 함.

view를 만드는 방법은 두가지이고
class를 활용하여 만드는 방법과
Function(함수형) View를 만드는 방법이 있다.

#----------------------------------------------

# 3. myvenv

> python -m venv myvenv
를 통해 만든 가상환경 폴더

중요한 폴더이고, 장고뿐만 아니고 다른 파이썬 프로젝트에서도 많이 사용 됨.
